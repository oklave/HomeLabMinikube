# Home Lab: Minikube on Proxmox (8 GB RAM Optimized)

> Однонодовый кластер Kubernetes для обучения, развёрнутый на домашнем сервере Proxmox. Оптимизирован под скромные ресурсы (8 ГБ ОЗУ, 4 vCPU) с использованием Minikube, Docker-драйвера и Ansible.

Здесь задокументирован полный процесс развёртывания учебного minikube:
- Подготовка ВМ в Proxmox
- Идемпотентная автоматизация через Ansible
- Настройка сети, хранилища и контейнерного рантайма
- Запуск Minikube с учётом ограничений памяти
- Диагностика, оптимизация и дальнейший roadmap обучения

## Архитектура и ограничения

| Компонент | Значение | Почему так |
|-----------|----------|------------|
| Хост | Proxmox VE 8.x | Тип 1 гипервизор, минимальный оверхед |
| ВМ | Ubuntu 24.04 LTS Server | Стабильный, cloud-init ready, актуальные пакеты |
| vCPU / RAM ВМ | 2 / 4 ГБ | Оставляет ~1–1.5 ГБ хосту Proxmox и кэш дисков |
| Minikube RAM | `--memory=3072` | 3 ГБ для контрольной плоскости + 1 ГБ ОС внутри ВМ |
| Драйвер | `docker` | Легче, чем `kvm2`/`qemu`, нет вложенной виртуализации |
| CNI | `calico` | Надёжный, хорошо работает на одноноде, меньше фоновых процессов |
| Автоматизация | Ansible | Идемпотентность, обработка прав групп, retry-логика |
